# Elitextra Dynamic Website Roadmap

## Working Rule

This repository is the publishing home for the Elitextra website. Source material may be read from `projects/Elitextra_Project`, but website code should be edited here only unless explicitly approved.

## Direction

Build a premium, warm, Nigerian agricultural export website that feels alive without becoming gimmicky. The motion should support buyer trust: product clarity, sourcing confidence, packaging quality, and fast inquiry flow.

## Phase 1: Stabilize This Repo

- Keep the current static HTML/CSS/JS site as the safe baseline.
- Commit the current site as the initial independent repo state.
- Exclude `_archive/`, local env files, dependency folders, and build output from publishing.
- Review all pages for broken links, weak copy, and missing metadata.
- Maintain a publishable static baseline before any framework migration.

## Phase 2: Brand Intelligence Pass

- Read brand foundation, brand DNA, design spec, product flyers, and product reference images as source material.
- Replace generic copy with direct-from-farm, traceability, quality, partnership, and market-specific messaging.
- Add product-specific content for cocoa, dry pap, beans powder, palm oil, hibiscus, and garri.
- Keep unconfirmed claims out of final copy until verified by the user.

## Phase 3: Dynamic Static Upgrade

- Add scroll storytelling sections using vanilla CSS/JS first.
- Add a richer buyer-path planner for sample, wholesale, and recurring supply.
- Add product detail drawers, quote persistence, interactive route cards, and animated product comparisons.
- Add progressive enhancement fallbacks so the site still works without advanced effects.

## Phase 4: Asset Upgrade

- Generate or source local campaign visuals for hero, product range, farm-to-port, and export desk sections.
- Prefer real product/reference assets where available.
- Use AI-generated images only when they improve brand credibility and are saved into `assets/generated/`.
- Avoid linking production pages to temporary external image URLs.
- Create a social share image before the first serious client-facing publish.

## Phase 5: Framework Decision

Move to Next.js only if the desired experience requires:

- Framer Motion page transitions and scroll choreography.
- Real product detail routes.
- Sample shop/cart.
- API-backed quote/contact forms.
- Spline, Three.js, or React Three Fiber scenes.

If the static site can achieve the needed result, keep it static for speed and deployment simplicity.

## Phase 6: Dynamic Premium Build

- Add cinematic hero with product/export motion.
- Add animated sourcing route from farm clusters to Lagos export coordination.
- Add product-specific pages or sections with richer buying logic.
- Add market pages for USA, UK, Europe, and Nigeria when copy is ready.
- Add real form handling before publication.

## Publish Milestones

### Milestone A: Credible Public Presence

- Static site is polished and deployable.
- Inquiry path is clear.
- No fake contact, pricing, certification, or volume claims.
- Client can visit a real URL.

### Milestone B: Client-Impress Showcase

- Stronger generated/local campaign imagery.
- Market-specific sections.
- Product-specific story and packaging content.
- More polished transitions and scroll storytelling.

### Milestone C: World-Level Dynamic Build

- Framework migration if needed.
- Framer Motion, Three.js/Spline, video loops, sample shop, real forms, and analytics.

## Asset Requests For User

- Best current product/flyer images for cocoa, dry pap, beans powder, palm oil, hibiscus, and garri.
- Any preferred background video loops from Higgsfield, Meta, Runway, Pika, or similar.
- Optional Spline links or 3D model files for product packaging or commodity objects.
- Confirmation before installing Next.js, Tailwind, Framer Motion, Three.js, or any dependency.
